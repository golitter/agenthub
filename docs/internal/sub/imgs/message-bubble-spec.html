重构方案 · session 管理抽离

现状：会话状态、消息流、导航状态耦合在 stores/chat.ts（barrel re-export），随会话数增长，单 store 膨胀、重渲染范围过大。

目标：按职责拆为三个独立 store —— navigation（currentSessionId + activeTab）、session（各会话独立数据 Map）、message（消息流式更新 + runtime blocks）。

步骤：

① 新建 session-store.ts，迁移 sessions Map 与按 sessionId 取数的 selector；② 新建 message-store.ts，承载 SSE 流式 block 归约（block-reducer）与滚动锚定；③ navigation-store.ts 保留 currentSessionId / activeTab，作为路由单一来源；④ chat.ts 降级为纯 barrel re-export，对外 API 不变，保证调用点零改动。

风险：selector 粒度需收紧（按 sessionId 切片），否则 Map 引用变更仍触发全量重渲染；流式高频更新需配合 useMessageScroll 的锚定策略，避免抖动。

验证：契约层 make generate 后三端类型对齐；前端用 __tests__ 覆盖 store 拆分后的隔离性与 selector 稳定性；E2E 跑多 Agent 群聊流式场景确认无回归。

回滚：保留旧 chat.ts 一版作为 fallback，feature flag 控制新旧 store 切换，灰度无异常后删除旧实现。

排期：P0 拆分（2d）→ P1 selector 优化（1d）→ P2 测试与灰度（1d），合计约 4 人日。
